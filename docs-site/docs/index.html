Documentation

Complete guide to integrating with Execution Protocol for AI agent validation and execution.

Overview

Execution Protocol provides infrastructure for AI agents to validate opportunities, execute trades, and build onchain reputation. It's designed specifically for the agent economy with deterministic security, flexible fees, and Base chain optimization.

Quick Start

1. Validate an Opportunity

curl -X POST https://api.execution-protocol.io/api/v1/validate \
  -H "Content-Type: application/json" \
  -d '{
    "agent_id": "my-agent-001",
    "opportunity": {
      "type": "hyperliquid_funding",
      "asset": "ETH",
      "expected_return": 0.05,
      "confidence": 0.90,
      "max_capital": 5000
    }
  }'

2. Execute Approved Trade

curl -X POST https://api.execution-protocol.io/api/v1/execute \
  -H "Content-Type: application/json" \
  -d '{
    "decision_id": "d-abc123",
    "approval_token": "APPROVE:d-abc123:EXEC:nonce123"
  }'

Execution Protocol uses a smart fee calculation:

Fee = max(0.5% of trade value, 0.01 ETH)

Security

Approval Tokens

Every execution requires a deterministic approval token:

Format: APPROVE:{decision_id}:EXEC:{nonce}

Example: APPROVE:d-1772954413261:EXEC:abc123

Onchain Attestation

Every decision is recorded on Base Sepolia for immutable audit trails:

  • Decision ID and timestamp
  • Confidence score
  • Status (approved/rejected)
  • Max allocation
